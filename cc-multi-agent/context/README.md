# context ディレクトリ

プロジェクト固有のコンテキストを管理するディレクトリ。

## 目的
- プロジェクトごとの知識・決定事項を保存
- セッション間での情報共有
- 新規参加者（employee）への引継ぎ

## global_context.md との使い分け

| ファイル | 用途 | 更新者 |
|----------|------|--------|
| `memory/global_context.md` | stockholder の好み、プロジェクト横断の知見 | president |
| `context/{project_id}.md` | 特定プロジェクトの詳細情報 | boss（タスク完了時） |

**判断基準:**
- 他のプロジェクトでも使える知見 → `global_context.md`
- このプロジェクト固有の情報 → `context/{project_id}.md`

## 使い方

### 新規プロジェクト追加時
1. `context/{project_id}.md` を作成
2. 下記テンプレートに沿って記載

### 作業開始時
1. `memory/global_context.md` を読む（システム全体の設定）
2. `context/{project_id}.md` を読む（プロジェクト固有情報）

### 更新時
- **更新責任者**: boss（タスク完了報告を受けた時）
- **更新タイミング**: 重要な決定事項、ブロッカー解消時、フェーズ完了時
- ファイルが存在しない場合は、boss が新規作成

## テンプレート

新規プロジェクトファイルは以下の形式で作成してください:

---

# {project_id}
最終更新: YYYY-MM-DD

## What（これは何か）
{1-2文で説明}

## Why（なぜやるのか）
- 目的:
- 成功の定義:

## Who（誰が関係するか）
- 責任者:
- 関係者:

## Constraints（制約）
- 期限:
- 予算:
- その他:

## Current State（今どこにいるか）
- ステータス: {未着手/進行中/レビュー中/完了}
- 進捗:
- 次のアクション:
- ブロッカー:

## Decisions（決まったこと）
| 日付 | 決定事項 | 理由 |
|------|----------|------|

## Notes（メモ）
{自由記述}
